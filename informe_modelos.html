3. MODELADO DE FUNCIONES — Diagramas de Secuencia UML

3.1 FUNCIÓN 1: Envío y Recepción de Mensaje de Texto

Escenario: Usuario A escribe "Hola" y lo envía a Usuario B. Ambos ya están conectados.

sequenceDiagram
    participant UA as Cliente A — EDT Swing
    participant SOA as ObjectOutputStream A
    participant MA as ManejadorCliente A — Hilo Servidor
    participant SRV as Servidor — HashMap synchronized
    participant MB as ManejadorCliente B — Hilo Servidor
    participant SOB as ObjectOutputStream B
    participant HRB as HiloReceptor B
    participant UB as Cliente B — EDT Swing

    UA->>UA: 1. Selecciona "Usuario B" en JList y escribe "Hola"
    UA->>UA: 2. Crea Mensaje(emisor=A, receptor=B, tipo=TEXTO, contenido=Hola)
    UA->>SOA: 3. synchronized(salida) writeObject(msg) flush() reset()
    Note over SOA: MARSHALLING: Objeto Mensaje se convierte a bytes
    SOA-->>MA: 4. Bytes viajan por TCP/LAN
    MA->>MA: 5. entrada.readObject() — deserialización
    Note over MA: UNMARSHALLING: bytes se reconstruyen como Mensaje
    MA->>MA: 6. switch(tipo) -> case TEXTO
    MA->>SRV: 7. Servidor.reenviarMensaje(msg)
    SRV->>SRV: 8. synchronized(clientesConectados) busca "B" en HashMap
    SRV->>MB: 9. destino.enviarMensaje(msg)
    MB->>SOB: 10. synchronized(salida) writeObject(msg) flush() reset()
    Note over SOB: MARSHALLING: Objeto -> bytes nuevamente
    SOB-->>HRB: 11. Bytes viajan por TCP/LAN
    HRB->>HRB: 12. entrada.readObject() — deserialización
    HRB->>HRB: 13. switch -> case TEXTO
    HRB->>UB: 14. SwingUtilities.invokeLater() areaMensajes.append("A: Hola")
    Note over UB: Se muestra en pantalla de forma thread-safe

Detalles técnicos de cada paso crítico:

3
synchronized(salida) en el Cliente: Protege el ObjectOutputStream porque el hilo EDT podría intentar enviar mientras otro envío aún no termina. reset() limpia la caché interna del stream para evitar que Java envíe referencias obsoletas en lugar del objeto actualizado.
8
synchronized(clientesConectados): Garantiza exclusión mutua. Si otro cliente se conecta al mismo instante, ambas operaciones no pueden tocar el HashMap simultáneamente, evitando condiciones de carrera.
14
SwingUtilities.invokeLater(): El HiloReceptor NO puede modificar componentes Swing directamente (no es thread-safe). Este método encola la actualización para que la ejecute el hilo EDT.

3.2 FUNCIÓN 2: Envío y Recepción de Archivo Multimedia

Escenario: Usuario A selecciona un archivo PDF de 2 MB y lo envía a Usuario B.

sequenceDiagram
    participant UA as Cliente A — EDT Swing
    participant FS as FileSystem Disco de A
    participant SOA as ObjectOutputStream A
    participant MA as ManejadorCliente A — Hilo Servidor
    participant SRV as Servidor — HashMap synchronized
    participant MB as ManejadorCliente B — Hilo Servidor
    participant SOB as ObjectOutputStream B
    participant HRB as HiloReceptor B
    participant FSB as FileSystem Disco de B
    participant UB as Cliente B — EDT Swing

    UA->>UA: 1. Selecciona "Usuario B" y presiona boton Archivo
    UA->>UA: 2. JFileChooser.showOpenDialog() selecciona informe.pdf
    UA->>UA: 3. Validacion: archivo.length() debe ser menor a 10 MB
    UA->>FS: 4. FileInputStream.read(datosArchivo)
    FS-->>UA: 5. byte[2097152] — 2 MB cargados en memoria
    UA->>UA: 6. Crea Mensaje(emisor=A, receptor=B, nombre=informe.pdf, datos=byte[2MB])
    Note over UA: El constructor asigna tipo=ARCHIVO automaticamente
    UA->>SOA: 7. synchronized(salida) writeObject(msg) flush() reset()
    Note over SOA: MARSHALLING PESADO: Objeto + 2MB de bytes serializados juntos
    SOA-->>MA: 8. Bytes viajan por TCP/LAN fragmentados por TCP
    MA->>MA: 9. readObject() deserializa Mensaje completo con los 2 MB
    MA->>MA: 10. switch -> case ARCHIVO, log en consola
    MA->>SRV: 11. Servidor.reenviarMensaje(msg)
    SRV->>SRV: 12. synchronized(clientesConectados) busca "B"
    SRV->>MB: 13. destino.enviarMensaje(msg)
    MB->>SOB: 14. synchronized(salida) writeObject(msg) flush() reset()
    SOB-->>HRB: 15. Bytes viajan por TCP/LAN
    HRB->>HRB: 16. readObject() deserializa Mensaje con datosAdjuntos
    HRB->>HRB: 17. switch -> case ARCHIVO llama recibirArchivo(msg)
    HRB->>FSB: 18. Crea carpeta archivos_recibidos/ con mkdirs()
    HRB->>FSB: 19. FileOutputStream.write(datosAdjuntos) guarda informe.pdf
    FSB-->>HRB: 20. Archivo guardado exitosamente
    HRB->>UB: 21. SwingUtilities.invokeLater() muestra confirmacion en pantalla

Detalles técnicos de cada paso crítico:

4-5
Lectura completa en memoria: El archivo se carga entero en un byte[]. Java serializa los datos binarios junto con el resto del objeto Mensaje de forma atómica (todo o nada).
6
Constructor especial: Cuando se usa Mensaje(emisor, receptor, nombreArchivo, datosAdjuntos), el tipo se asigna automáticamente como ARCHIVO en la línea 91 de Mensaje.java.
8
Fragmentación TCP: Aunque el objeto serializado pese megabytes, TCP se encarga automáticamente de fragmentarlo en paquetes y reensamblarlos en orden. Transparente para el programador.
18
Creación de directorio: El receptor crea automáticamente archivos_recibidos/ con File.mkdirs() si no existe.
